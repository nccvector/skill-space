# Output Format

Return a clear research result, not a pile of half-digested notes.

## Required Section Order

Use this order unless the user asked for a different artifact:

1. `Summary`
2. `Key findings` or `Options`
3. `Open questions`
4. `Residual uncertainty`

If the task is a comparison, `Options` may replace `Key findings`.

## Summary

The summary should:

- answer the main question directly
- stay evidence-backed
- mention recency if it matters

## Key Findings Or Options

Use this section for:

- main facts
- meaningful comparisons
- tradeoffs
- notable risks
- high-signal source-backed observations

Keep it compact and specific.

## Open Questions

Use this section for:

- unresolved issues
- missing evidence
- local decisions still needed from the user or team

Keep it short.

## Residual Uncertainty

Use this section to state what remains unclear and why:

- conflicting sources
- limited recency
- incomplete public information
- no live web search available

Good research can still end with uncertainty. Just label it honestly.

## Citation And Attribution Behavior

When sources matter to the usefulness of the answer:

- attribute facts to their source category or document
- mention concrete dates for time-sensitive claims
- distinguish sourced facts from your own inference

The output should be useful to a human who may want to verify the trail later.

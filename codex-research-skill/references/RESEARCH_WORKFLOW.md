# Research Workflow

The goal is thorough, evidence-backed research.
Default to a research mindset: gather evidence first, compare claims, separate facts from inference, and keep searching until the question is well-covered.

## Scope The Research

Determine what kind of research this is:

- current-state lookup
- option comparison
- technical deep dive
- decision support
- risk investigation

If the scope is genuinely ambiguous and the choice would change the output meaningfully, ask via `AskUserQuestion`.

## Gather Evidence

When search is available:

- run multiple searches
- refine queries based on what you learn
- look for primary and official sources first
- compare evidence across sources
- continue until you can explain the topic with confidence and caveats

When local repo context matters:

- read the relevant docs, code, or existing notes
- connect external findings to the local problem

## Evidence Standards

Prefer evidence that is:

- primary
- current
- specific
- attributable

Be careful with:

- marketing summaries
- low-signal blog posts
- forum anecdotes
- stale docs

Use them only when they add value and do not outrank better sources.

## Synthesis Rules

Separate:

- what the sources say
- what you infer from the sources
- what remains uncertain

If the evidence conflicts, say so.
If the evidence is thin, say so.

## When To Stop Searching

Stop when:

- the main question is answered with adequate evidence
- major alternative explanations or options have been checked
- remaining uncertainty is narrow and clearly stated

Do not stop after one decent result if the topic is recent, contested, or important.

## Fallback Path

If web search is unavailable:

- continue with repo materials and stable knowledge
- clearly say that external verification was not available
- lower confidence accordingly

Research without search can still be useful, but it should not pretend to be fresh.
